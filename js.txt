Chapter 4 – Control Flow (Decision Making in JavaScript)

Goal of this Chapter

By the end of this chapter, you should understand how JavaScript decides which code to execute and which code to skip.

Think of Control Flow as the brain of your program.

Variables store data.

Operators perform operations.

Control Flow makes decisions.

The concepts in this chapter cover if, else if, else, switch, truthy/falsy conditions, and the early return pattern.

1. What is Control Flow?

Imagine you're writing code for an ATM.

If balance is enough

↓

↓

Show "Insufficient Balance"

How does JavaScript know which code to run?

Answer:

Control Flow

Which statement runs

Which statement is skipped

How many times something runs

Think of it like

Traffic Signal

Green → Go

Yellow → Wait

Exactly the same happens in JavaScript.

🎯 Interview Questions

Difference between for and while?

for is preferred when the number of iterations is known. while is better when the stopping condition depends on something that may change unpredictably.

Difference between break and continue?

break exits the loop completely.

continue skips only the current iteration.

Difference between for...of and for...in?

for...of iterates over values.

for...in iterates over keys (property names).

Why can't break be used inside forEach()?

Because forEach() is a method, not a loop statement. It doesn't provide the control flow needed for break or continue.

💼 Professional Tips

✔ Prefer const inside loops whenever the loop variable itself doesn't need reassignment.

✔ Use meaningful variable names (student, product) instead of always using i when iterating values.

✔ Avoid deeply nested loops when possible—they can make code harder to read and slower for large datasets.

✔ Choose the loop that matches the problem instead of forcing one style everywhere.

### Why are `shift()` and `unshift()` slower?

Because all remaining elements need to change their positions.

---

### What is `length`?

The total number of elements in the array.

---

# 💼 Professional Tips

✔ Use arrays when the **order of data matters**.

✔ Don't hardcode indexes if you can use loops.

✔ Prefer `push()` over `unshift()` when possible for better performance.

✔ When accessing the last element, always write

```js
arr[arr.length - 1]
```

instead of guessing the index.

# 🎯 Interview Questions

### Difference between `map()` and `forEach()`?

* `map()` returns a **new array**.
* `forEach()` is mainly for performing actions (printing, updating external state) and returns `undefined`.

---

### Difference between `filter()` and `find()`?

* `filter()` returns **all matching elements**.
* `find()` returns **only the first matching element**.

---

### Difference between `some()` and `every()`?

* `some()` → At least one element satisfies the condition.
* `every()` → Every element must satisfy the condition.

---

### Why is `reduce()` useful?

It combines multiple values into a single result such as a sum, average, count, or grouped object.

---

# 💼 Professional Tips

✔ Use `map()` when transforming data.

✔ Use `filter()` when selecting data.

✔ Use `find()` when only one item is needed.

✔ Use `reduce()` for calculations and aggregations.

✔ Keep callback functions short and readable.

# ⭐ 1. `slice()` ⭐⭐⭐⭐⭐

## Definition

`slice()` **copies a portion of an array** and returns a **new array**.

> **It NEVER changes the original array.**

---

## Syntax

```js
array.slice(start, end)
```

* `start` → Included
* `end` → Excluded

Think of it as:

```text
[start, end)
```

Start is included.

End is **not**.

---

## Example 1

```js
let arr = [10,20,30,40,50];

let ans = arr.slice(1,4);

console.log(ans);
```

Output

```text
[20,30,40]
```

---

### Memory Diagram

Before

```text
Index

0    1    2    3    4
┌───┬───┬───┬───┬───┐
│10 │20 │30 │40 │50 │
└───┴───┴───┴───┴───┘
```

```text
slice(1,4)
```

Copies

```text
20

30

40
```

Original array

```text
10 20 30 40 50
```

still remains unchanged.

---

## Example 2

```js
arr.slice(2);
```

Output

```text
[30,40,50]
```

Meaning

From index **2** till the end.

---

## Example 3

Negative Index

```js
arr.slice(-2);
```

Output

```text
[40,50]
```

Negative indexing starts counting from the end.

---

# ⭐ Important Rule

```text
slice()

↓

Copies

↓

Returns New Array

↓

Original Safe ✅
```


# ⭐ 2. `splice()` ⭐⭐⭐⭐⭐

## Definition

`splice()` **changes the original array**.

It can:

* Remove elements
* Insert elements
* Replace elements

Think of it as the **Swiss Army Knife** of array modification.

---

## Syntax

```js

```

or

```js

```

---

## Remove Example

```js

arr.splice(1,2);
```

---

Before

```text
10 20 30 40
```

After

```text
10 40
```

Because

Started at index 1

Deleted 2 elements.

---

## Insert Example

```js
let arr = [10,40];

arr.splice(1,0,20,30);
```

Output

```text
10

20

30

40
```

Notice

Deleted

```text

```

Inserted

```text
20

30
```

---

## Replace Example

```js
let arr=[10,20,30];

arr.splice(1,1,100);
```

Output

```text
10

100

30
```

# ⭐ 3. Array Destructuring ⭐⭐⭐⭐⭐

## Why?

Suppose

```js
let arr = ["Viraj",20,"Pune"];
```

Without destructuring

```js
let name = arr[0];
let age = arr[1];
let city = arr[2];
```

Too much work.

---

## Solution

```js
let [name,age,city]=arr;
```

Done.

---

## Internal Working

```text
Array

↓

["Viraj",20,"Pune"]

↓

name → Viraj

age → 20

city → Pune
```

---

## Skip Values

```js
let arr=[10,20,30];

let [a,,c]=arr;
```

Output

```text
a = 10

c = 30
```

20 ignored.

---

## Default Values

```js
let arr=[10];

let [a,b=100]=arr;
```

Output

```text
10

100
```

# ⭐ 4. Spread Operator (`...`) ⭐⭐⭐⭐⭐

Definition

The spread operator **expands** an array into individual values.

---

Example

```js
let arr=[10,20,30];

console.log(...arr);
```

Output

```text

```

---

## Copy Array

```js
let arr=[1,2,3];

let copy=[...arr];
```

New array created.

---

## Merge Arrays

```js
let a=[1,2];

let b=[3,4];

let c=[...a,...b];
```

Output

```text
1

2

3

4
```

---

## Why not

```js
let b = a;
```

Because

Both variables point to the same array.

Changing one changes the other.

Spread creates a new copy.

# 🌍 Why Do Objects Exist?

Imagine you want to store information about a student.

Without objects:

```js
let name = "Viraj";
let age = 20;
let city = "Pune";
let cgpa = 8.42;
```

Now imagine 500 students...

😵 Chaos.

Instead, group everything together.

```js
const student = {
    name: "Viraj",
    age: 20,
    city: "Pune",
    cgpa: 8.42
};
```

Everything related to one student lives inside **one object**.

---

# 📖 Definition

An **Object** is a collection of **key-value pairs**.

```text
Key  → Value

name → Viraj

age  → 20

city → Pune
```

Think of an object as a **real-world entity**.

Examples:

```text

↓

━━━━━━━━━━━━

Product

↓

Title
Price
Rating
Stock
```

```text
OBJECT
```

Not separate variables.

# 11. Copying Objects ⭐⭐⭐⭐⭐

## Wrong Way

```js
let b = person;
```

Both variables point to the **same object**.

```text
person ──┐
         ▼
      Object
         ▲
b ───────┘
```

Change one...

Both change.

---

## Correct Way (Spread)

```js
let copy = {...person};
```

Creates a **shallow copy**.

---

## Object.assign()

```js
let copy = Object.assign({}, person);
```

Also creates a shallow copy.

---

# Shallow Copy vs Deep Copy ⭐⭐⭐⭐⭐

Suppose

```js
const student = {
    name: "Viraj",
    address: {
        city: "Pune"
    }
};
```

Spread copies only the first level.

The nested object is still shared.

```text
student.address

↓

Same Object

↓

copy.address
```

---

## Deep Clone

Modern JavaScript

```js
const copy = structuredClone(student);
```

Everything is copied independently.

# 🎯 Interview Questions

### Difference between dot and bracket notation?

* Dot → Fixed property names.
* Bracket → Dynamic property names.

---

### Difference between shallow copy and deep copy?

* Shallow copy copies only the first level.
* Deep copy duplicates nested objects too.

---

### Why use optional chaining?

To safely access nested properties without throwing errors.

---

### Difference between `Object.keys()` and `Object.entries()`?

* `Object.keys()` → Returns only property names.
* `Object.entries()` → Returns `[key, value]` pairs.

---

### Why is object assignment dangerous?

Because it copies the **reference**, not the actual object.

# 💼 Professional Tips

✔ Use `const` for objects. You can still modify their properties.

✔ Prefer object destructuring to reduce repetitive code.

✔ Use optional chaining with API data.

✔ Use spread for shallow copies.

✔ Use `structuredClone()` when you truly need a deep copy.

# 🏆 Chapter Summary

If you remember these points, you'll have a solid grasp of objects:

* ✅ Objects store **key-value pairs** describing one entity.
* ✅ Use **dot notation** for fixed keys and **bracket notation** for dynamic keys.
* ✅ Objects can contain nested objects.
* ✅ Destructuring extracts properties into variables.
* ✅ `for...in`, `Object.keys()`, and `Object.entries()` are your main tools for traversal.
* ✅ Spread (`...`) and `Object.assign()` create **shallow copies**.
* ✅ `structuredClone()` creates a **deep copy**.
* ✅ Optional chaining (`?.`) prevents errors when accessing missing nested properties.
* ✅ Computed properties let you create dynamic property names.
* ✅ Objects are reference types—assignment copies the reference, not the object.
